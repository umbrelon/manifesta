## table.json — top-level properties

| Property | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `name` | string | Yes | — | Qualified table name (e.g. `"dbo.Customer"`). Must be unique across the registry. |
| `fields` | FieldDefinition[] | Yes | — | Column definitions. Order is preserved in generated documentation. |
| `primaryKey` | string[] | No | `[]` | Ordered list of field names that form the primary key. Each entry must match a `name` in `fields`. |
| `description` | string | No | — | Full description of the table rendered as a bullet point below the table heading in `database.md`. |
| `shortDescription` | string | No | — | One-liner (≤15 words) shown after the table name in the Table of Contents. Omitted from JSON when empty. |
| `labelField` | string | No | — | Name of the field that holds the human-readable display value for this table. Used by documentation generators and full-edition AI features. Declared once on the referenced table, not on individual FKs. |
| `isReferenceTable` | boolean | No | `false` | Marks this as a lookup/reference table. Enables the `data` property for embedding row values. |
| `data` | object[] | No | — | Row data. Each object maps column names to values. Only written when `isReferenceTable: true`. See [Reference Tables](./schema-features.md#reference-tables) for validation rules. |
| `isDeprecated` | boolean | No | `false` | Marks the table as deprecated. Adds a `[DEPRECATED]` badge and blockquote notice in `database.md`. |
| `deprecationMessage` | string | No | — | Shown as the body of the deprecation notice. Only rendered when `isDeprecated: true`. |
| `foreignKeys` | ForeignKey[] | No | `[]` | FK relationships originating from this table. |
| `uniqueConstraints` | object[] | No | `[]` | Unique constraints. Populated automatically by `init db` and `init prisma`. |
| `indexes` | object[] | No | `[]` | Index definitions. Populated automatically by `init db` and `init prisma`. |
| `checkConstraints` | object[] | No | `[]` | CHECK constraints. Populated automatically by `init db` (SQL Server full edition). |

## FieldDefinition properties

| Property | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `name` | string | Yes | — | Column name. Must be unique within the table. |
| `type` | string | Yes | — | SQL type as a string (e.g. `"int"`, `"nvarchar(100)"`, `"decimal(10,2)"`). |
| `nullable` | boolean | Yes | — | Whether the column allows NULL values. |
| `isPrimaryKey` | boolean | No | `false` | Marks this field as part of the primary key. Also add the field name to the table-level `primaryKey[]`. |
| `description` | string | No | — | Column description rendered in the field table in `database.md`. |
| `descriptionAutoGenerated` | boolean | No | `false` | When `true`, the field description was generated by the full edition's `ai describe` command and has not yet been reviewed. Causes the table heading to render with a trailing asterisk (`*`) in `database.md` until all fields are accepted. |
| `defaultValue` | string | No | — | Column default value. Populated by `init prisma` from `@default(...)` annotations. |
| `isComputed` | boolean | No | `false` | `true` when this is a computed/generated column. |
| `computedExpression` | string | No | — | The SQL expression for computed columns (e.g. `"([FirstName]+' '+[LastName])"`). Required when `isComputed: true`. |
| `isPersisted` | boolean | No | `false` | `true` when the computed column is physically stored (`PERSISTED`). Only meaningful when `isComputed: true`. |
| `isMatchColumn` | boolean | No | `false` | Marks this field as a match/lookup column. Cannot be nullable — validated as `MATCH-COLUMN-NULLABLE`. |
| `isDeprecated` | boolean | No | `false` | Marks this field as deprecated. Rendered with strikethrough in `database.md`. |
| `deprecationMessage` | string | No | — | Shown alongside the strikethrough in the field table. Only rendered when `isDeprecated: true`. |
| `sensitivity` | string | No | — | Sensitivity classification. Valid values (case-insensitive): `"PII"`, `"Confidential"`, `"Internal"`, `"Public"`. Adds a Sensitivity column to the field table: 🔴 PII, 🟡 Confidential, 🔵 Internal, 🟢 Public. |

## ForeignKey properties

| Property | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `sourceField` | string | Yes | — | Field name on this table that holds the FK value. Must exist in `fields`. |
| `targetTable` | string | Yes | — | Qualified name of the referenced table (e.g. `"dbo.Customer"`). Checked by `validate cross`. |
| `targetField` | string | Yes | — | Field name on the target table being referenced. |
| `kind` | string | No | `"physical"` | FK kind. `"physical"` — enforced by the database; detected by full-edition introspection. `"logical"` — business-logic relationship, not enforced at DB level; never removed by automated tooling. `"virtual"` — documentation-only link; excluded from ERDs and topological sort by default. |
| `cascadeDelete` | boolean | No | `false` | Whether the FK has `ON DELETE CASCADE`. Only valid on physical FKs. |

> **Legacy:** `"soft": true` is accepted and silently mapped to `"kind": "logical"`. New files should use `kind` instead.

## section.json properties

| Property | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `name` | string | Yes | — | Section name. Must be unique across the registry. Used as the heading in `database.md` and in `sectionOrder` config. |
| `tables` | string[] | Yes | — | Ordered list of qualified table names belonging to this section. Each name must match a table in the registry — checked by `validate cross`. |
| `description` | string | No | — | Section description rendered below the section heading in `database.md`, before the ERD diagrams. |
| `erds` | ErdDefinition[] | No | `[]` | ERD diagrams to render for this section. Each entry produces one Mermaid diagram block. |
| `isModule` | boolean | No | — | When `true`, marks this section as an independently installable module. Used by the full edition's `db tenant-drift` command to scope drift detection per database. Omitted from JSON when not set. |

---

## ErdDefinition properties

| Property | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `title` | string | Yes | — | Diagram title rendered as bold text above the Mermaid block. |
| `tables` | string[] | No | section tables | Subset of the section's tables to include in this diagram. When omitted, all section tables are included. |
| `fields` | string | No | `"pk-and-fk"` | Field display verbosity inside entity boxes. `"pk-and-fk"` — PK and FK columns only. `"all"` — every column. `"none"` — entity names and relationships only. |
| `includeLogical` | boolean | No | `true` | Whether logical FKs are included in this diagram. |
| `includeVirtual` | boolean | No | `false` | Whether virtual FKs are included in this diagram. |

CLI flags `--no-logical` and `--include-virtual` on `manifesta doc db` override these per-ERD settings globally for the entire run.

## Complete table.json example

All properties shown — in practice most are omitted and resolved to their defaults.

```json
{
  "name": "dbo.Customer",
  "description": "Registered customers. Each row represents one customer account.",
  "shortDescription": "Customer accounts",
  "labelField": "Name",
  "isReferenceTable": false,
  "isDeprecated": false,
  "primaryKey": ["Id"],
  "fields": [
    {
      "name": "Id",
      "type": "int",
      "nullable": false,
      "isPrimaryKey": true,
      "description": "Surrogate primary key."
    },
    {
      "name": "Name",
      "type": "nvarchar(100)",
      "nullable": false,
      "description": "Display name of the customer."
    },
    {
      "name": "Email",
      "type": "varchar(255)",
      "nullable": false,
      "sensitivity": "PII",
      "description": "Primary contact email address."
    },
    {
      "name": "StatusId",
      "type": "int",
      "nullable": false,
      "description": "Current account status.",
      "isDeprecated": false
    },
    {
      "name": "FullName",
      "type": "nvarchar(201)",
      "nullable": true,
      "isComputed": true,
      "computedExpression": "([FirstName]+' '+[LastName])",
      "isPersisted": false,
      "description": "Computed display name."
    }
  ],
  "foreignKeys": [
    {
      "sourceField": "StatusId",
      "targetTable": "dbo.CustomerStatus",
      "targetField": "Id",
      "kind": "physical"
    }
  ],
  "uniqueConstraints": [
    { "fields": ["Email"] }
  ]
}
```

## Complete section.json example

```json
{
  "name": "Core",
  "description": "Core business entities — customers and their orders.",
  "tables": ["dbo.Customer", "dbo.Order", "dbo.CustomerStatus"],
  "erds": [
    {
      "title": "Overview",
      "fields": "none"
    },
    {
      "title": "Key relationships",
      "tables": ["dbo.Customer", "dbo.Order"],
      "fields": "pk-and-fk",
      "includeLogical": true,
      "includeVirtual": false
    },
    {
      "title": "Full detail",
      "fields": "all"
    }
  ]
}
```
