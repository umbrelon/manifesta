# Security Policy

## Reporting a vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security issues privately via [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) on this repository, or email the maintainers directly (see the repository contact).

Include:
- A description of the vulnerability and its potential impact
- Steps to reproduce or a minimal proof-of-concept
- Which version(s) are affected

You will receive an acknowledgement within 72 hours. We aim to triage and patch confirmed vulnerabilities within 14 days.

## Scope

This policy covers the Manifesta OSS CLI and its library projects (`Manifesta.Core`, `Manifesta.Doc`).

## Out of scope

- Vulnerabilities in third-party dependencies (report those upstream)
- Issues requiring physical access to the machine running Manifesta
- Social engineering attacks
